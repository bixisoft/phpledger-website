Get Started in Minutes

Clone the repository and start building your accounting system today.

Installation
bash 
    git clone https://github.com/rmak78/phpledger.git
Basic Usage
php 
    <?php
    require_once 'phpledger/autoload.php';

    $cashAccount = new Account('Cash', 'Asset');
    $salesAccount = new Account('Sales', 'Revenue');

    $transaction = new Transaction();
    $transaction->addEntry($cashAccount, 100, 'debit');
    $transaction->addEntry($salesAccount, 100, 'credit');
    $transaction->save();

    echo $cashAccount->getBalance(); //100
    ?>

Frequently Asked Questions

Common questions about PHP Ledger and double-entry accounting.

Is PHP Ledger production-ready?

PHP Ledger is used by Sutlej Solutions for their everyday accounting needs. While it's functional, we recommend thorough testing for your specific use case before production deployment.

Does it require a specific PHP framework?

No! PHP Ledger is framework-agnostic and can be integrated into any PHP application, whether you're using Laravel, Symfony, CodeIgniter, or plain PHP.

How do I persist data?

PHP Ledger uses MySQL for data persistence. The system includes database schemas and basic CRUD operations for accounts, transactions, and entries.

What's the license?

PHP Ledger is open-source software. Check the LICENSE file in the repository for specific terms. You're free to use, modify, and distribute the code.

How can I contribute?

Fork the repository, make your improvements, and submit a pull request. Bug reports and feature requests are welcome in the GitHub issues section.

Do you provide support?

As stated in the README, free support is not provided. However, the community often helps with questions and issues in the GitHub discussions.
